# PosterFlow

Self-hosted poster management for Plex/Kometa homelabs. Syncs artwork from community-maintained Google Drives, matches it to your Radarr/Sonarr/Plex libraries, normalizes filenames into Kometa asset layout, optionally re-borders the artwork, and uploads to Plex. Single-container deploy.

PosterFlow is heavily inspired by — and aims for byte-for-byte output compatibility with — [DAPS by Drazzilb](https://github.com/Drazzilb08/daps).

## Quick start

```yaml
services:
  posterflow:
    image: dweagle/posterflow:develop
    container_name: posterflow
    ports:
      - "8357:8000"
    volumes:
      - ./config:/config              # required: DB, rclone config, logs
      # - ./assets:/assets            # optional: Kometa assets directory
    environment:
      - PUID=1000
      - PGID=1000
      - TZ=America/New_York
    restart: unless-stopped
```

`docker compose up -d`, browse to `http://localhost:8357`, walk the wizard.

## Features at a glance

- **Drive Sync** — Subscribe to community presets (CL2K, MM2K) or add custom Google Drives. Curated list of 41 drives bundled with the project.
- **Poster Renamer** — Match downloaded posters to Plex/Radarr/Sonarr items by ID or normalized title, organize into Kometa asset layout.
- **Border Replacer** — Crop, repaste, resize to exact 1000×1500. Holiday-schedule color overrides.
- **Unmatched Assets** — Detect library items missing posters; report with TMDB links.
- **Plex Upload** — Push posters directly to Plex. Webhook-driven uploads from Radarr/Sonarr.
- **IDarr** — TMDB/TVDB/IMDB ID assignment and rename normalization for poster-maker assets.
- **Maker Tools** — TMDB Search with Photopea PSD editing, monitor pages for upcoming releases.
- **Scheduler** — APScheduler-backed, persistent across restarts, cron / hourly / daily / weekly / monthly / multi-day.
- **Live status** — Three WebSocket endpoints for job progress and log streaming.
- **After-job scripts** — Arbitrary scripts run on job completion. Configurable per hook point.
- **Discord notifications** — Per-feature toggles, success/failure filters, mentions.
- **Community Requests** — Supabase-backed user request workflow with Discord OAuth for makers.

## Reproducing the screenshots in this documentation

The screenshots in `docs/images/` were generated by `scratch/capture.mjs` against a locally-built image. To regenerate:

```bash
git clone https://github.com/dweagle/posterflow.git
cd posterflow
docker build -t posterflow:docs .

mkdir -p scratch/config scratch/assets scratch/idarr
docker run -d --name pf-docs -p 8357:8000 \
  -v $(pwd)/scratch/config:/config \
  -v $(pwd)/scratch/assets:/assets \
  -v $(pwd)/scratch/idarr:/idarr \
  -e PUID=1000 -e PGID=1000 -e TZ=UTC -e DEBUG=true \
  posterflow:docs

cd scratch
npm install playwright sharp
docker run --rm --network host -v $(pwd):/work -w /work \
  mcr.microsoft.com/playwright:v1.60.0-noble \
  node capture.mjs --base http://localhost:8357 --out ../docs/images
```

The script is re-runnable. Reset state by stopping the container, `rm -rf scratch/config`, and starting it again.
